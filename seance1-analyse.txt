--------------------------------------------------------------------------------
1. OBJECTIF PROJET
--------------------------------------------------------------------------------

Le projet ServiceHub est une plateforme de gestion de services permettant de
centraliser la relation client, le suivi des interventions et la facturation.

Contexte métier :
- Entreprise de services B2B nécessitant un outil de ticketing
- Besoin de traçabilité des demandes clients
- Automatisation de la facturation liée aux interventions
- Reporting pour le pilotage de l'activité

--------------------------------------------------------------------------------
2. BESOINS FONCTIONNELS IDENTIFIÉS
--------------------------------------------------------------------------------

GESTION DES UTILISATEURS :
- Authentification sécurisée (email/mot de passe)
- Gestion des rôles et permissions
- Profils utilisateurs (clients, techniciens, administrateurs)
- Activation/désactivation des comptes

GESTION DES TICKETS :
- Création de tickets avec référence unique (TKT-YYYYMMDD-XXXX)
- Attribution des tickets aux techniciens
- Suivi des statuts (ouvert, en cours, en attente, résolu, fermé)
- Gestion des priorités (basse, moyenne, haute, critique)
- Système de commentaires (internes et publics)
- Pièces jointes

FACTURATION :
- Génération de factures avec référence unique (INV-YYYYMMDD-XXXX)
- Lignes de facture liées aux tickets
- Calcul automatique TVA et totaux
- Suivi des statuts de paiement
- Enregistrement des paiements (CB, virement, chèque, espèces)

REPORTING :
- Statistiques sur les tickets (volume, temps de résolution)
- Suivi du chiffre d'affaires
- Tableaux de bord

4. ARCHITECTURE TECHNIQUE
--------------------------------------------------------------------------------

Stack technique identifiée :
- Backend : NestJS (Node.js/TypeScript)
- Base de données : PostgreSQL
- ORM : Prisma
- Conteneurisation : Docker / Docker Compose
- Architecture : Clean Architecture (Controllers, Services, Repositories)

Organisation du code :
/backend
  /src
    /users      -> Module Utilisateurs
    /tickets    -> Module Tickets
    /billing    -> Module Facturation
    /reporting  -> Module Reporting (à développer)
    /common     -> Utilitaires partagés (guards, filters, interceptors)
    /config     -> Configuration
    /prisma     -> Service Prisma
